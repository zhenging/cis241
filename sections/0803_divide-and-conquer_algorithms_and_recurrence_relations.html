EXAMPLE 1
Binary Search We introduced a binary search algorithm in Section 3.1. This binary search algorithm reduces the search for an element in a search sequence of size nnn to the binary search for this element in a search sequence of size n/2n/2n/2, when nnn is even. (Hence, the problem of size nnn has been reduced to one problem of size n/2n/2n/2.) Two comparisons are needed to implement this reduction (one to determine which half of the list to use and the other to determine whether any terms of the list remain). Hence, if f(n)f (n)f(n) is the number of comparisons required to search for an element in a search sequence of size nnn, then

f(n)=f(n/2)+2\begin{aligned} f(n) = f(n/2) + 2 \end{aligned} f(n)=f(n/2)+2​

Merge Sort The merge sort algorithm (introduced in Section 5.4) splits a list to be sorted with n items, where n is even, into two lists with n/2 elements each, and uses fewer than n comparisons to merge the two sorted lists of n/2 items each into one sorted list. Consequently, the number of comparisons used by the merge sort to sort a list of n elements is less than M(n), where the function M(n) satisfies the divide-and-conquer recurrence relation

M(n)=2M(n/2)+n\begin{aligned} M(n) = 2M(n/2) + n \end{aligned} M(n)=2M(n/2)+n​
